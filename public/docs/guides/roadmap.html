v1.2 - MCP Server Mode (October 2025)

Major Feature: Dual-Mode Architecture

Context Foundry now supports two execution modes, giving users flexibility based on their needs:

‚úÖ MCP Mode (Terminal-Based - Claude Desktop Integration Pending)

  • ‚úÖ MCP (Model Context Protocol) server implementation using FastMCP (complete and functional)
  • ‚úÖ Terminal-based MCP server works with API keys
  • ‚ö†Ô∏è Future: Claude Desktop integration blocked by lack of sampling support (would use subscription instead of API charges)
  • ‚úÖ Three MCP tools implemented: context_foundry_build, context_foundry_enhance, context_foundry_status
  • ‚úÖ foundry serve command to start MCP server
  • ‚ö†Ô∏è Tools return helpful error messages explaining current limitation
  • ‚úÖ Automatic configuration help with --config-help

üí≥ API Mode (Continues to Work)

  • ‚úÖ Standalone CLI operation via foundry build
  • ‚úÖ Direct Anthropic API integration
  • ‚úÖ Works with Python 3.9+
  • ‚úÖ Good for CI/CD and automation

üì¶ Two-Tier Dependency Architecture

  • ‚úÖ Base installation works with Python 3.9+ (requirements.txt)
  • ‚úÖ Optional MCP mode requires Python 3.10+ (requirements-mcp.txt)
  • ‚úÖ Graceful degradation - users only install what they need
  • ‚úÖ Clear error messages guiding users to correct setup

üèóÔ∏è Technical Implementation

  • ‚úÖ Created ClaudeCodeClient for MCP mode (mirrors ClaudeClient interface)
  • ‚úÖ Factory function get_claude_client() selects appropriate client
  • ‚úÖ MCP server with three tools for building, enhancing, and status checking
  • ‚úÖ Fixed package structure - added __init__.py to all package directories
  • ‚úÖ Removed misleading claude_cli_integration.py

üìö Comprehensive Documentation

  • ‚úÖ New INSTALL.md with installation guide, troubleshooting, and lessons learned
  • ‚úÖ New docs/MCP_SETUP.md with step-by-step MCP configuration
  • ‚úÖ Configuration template in config/claude_desktop_config.example.json
  • ‚úÖ Documented 8 key lessons learned from implementation
  • ‚úÖ Common issues section with 6 specific problems and solutions
  • ‚úÖ Verification checklists for both modes

üéì Lessons Learned

  1. Python version management is critical - two-tier architecture solves compatibility
  2. Package structure matters - find_packages() requires __init__.py files
  3. PATH management is non-trivial - document exactly where scripts install
  4. Editable installs can be tricky - explicit Python version helps
  5. Dependencies should be gradual - optional features = optional dependencies
  6. Error messages should be helpful - guide users with exact commands
  7. Documentation should match reality - write after debugging, not before
  8. Two modes require two strategies - shared core with different clients

Key Insight: Users discovered that the original "Claude CLI" mode still charged API fees, leading to this comprehensive dual-mode implementation that provides true free usage via MCP while maintaining API mode for automation.

Completed: October 4, 2025

v2.1.0 - Enhancement Mode (October 2025)

Major Feature: Fix, Enhance, and Upgrade Existing Codebases

Context Foundry can now intelligently modify existing projects instead of only building from scratch:

‚úÖ Multi-Mode Support

  • ‚úÖ new_project - Build from scratch (original functionality)
  • ‚úÖ fix_bug - Fix bugs in existing code
  • ‚úÖ add_feature - Add features to existing code
  • ‚úÖ upgrade_deps - Upgrade dependencies
  • ‚úÖ refactor - Refactor existing code
  • ‚úÖ add_tests - Add tests to existing code

‚úÖ Intelligent Detection System

  • ‚úÖ Automatic codebase detection (15+ project types supported)
    • Python (requirements.txt, setup.py, pyproject.toml)
    • Node.js (package.json, tsconfig.json)
    • Rust (Cargo.toml), Go (go.mod), Java (pom.xml)
    • Ruby (Gemfile), PHP (composer.json), .NET (*.csproj)
    • And more...
  • ‚úÖ Intent detection from natural language keywords
    • "fix" ‚Üí fix_bug, "add" ‚Üí add_feature, "upgrade" ‚Üí upgrade_deps
  • ‚úÖ Auto-mode adjustment with conflict warnings
  • ‚úÖ Git repository status checking
  • ‚úÖ Confidence scoring (high/medium/low)

‚úÖ Phase 0: Codebase Analysis (NEW!)

  • ‚úÖ Runs ONLY for enhancement modes (skipped for new projects)
  • ‚úÖ Analyzes project structure, architecture, and existing tests
  • ‚úÖ Reviews git history and current branch state
  • ‚úÖ Creates codebase-analysis.md report for context
  • ‚úÖ Provides full understanding before making changes

‚úÖ Enhancement-Aware Orchestrator

  • ‚úÖ Scout phase: Mode-specific analysis strategies
    • Target ed bug finding for fix_bug
    • Integration point analysis for add_feature
    • Dependency impact assessment for upgrade_deps
  • ‚úÖ PHASE 2.5 (Parallel Builders): Targeted modifications
    • Modifies existing files instead of creating new projects
    • Preserves existing code structure and patterns
    • Creates feature branches before making changes
    • Groups changes by logical components
  • ‚úÖ Test phase: Validates existing functionality preserved
  • ‚úÖ Deploy phase: Feature branch + Pull Request workflow
    • Pushes to feature branch (NOT main)
    • Creates PR with detailed description
    • Links to GitHub issues (Closes #N)
    • Requires human review before merge

‚úÖ Testing & Validation

  • ‚úÖ Standalone test script (test_detection.py)
  • ‚úÖ Live test: YouTube Transcript Summarizer enhancement
    • Added auto-save markdown feature
    • 21 new tests (all passing)
    • Created PR #2 on feature branch
    • Zero impact on existing functionality
    • Completed in ~12 minutes, 1 test iteration

üìä Impact

  • üéØ Codebase Coverage: Detects Python, JavaScript/TypeScript, Rust, Go, Java, Ruby, PHP, .NET, C/C++, and more
  • üöÄ Success Rate: 100% on first live test (YouTube Transcript Summarizer)
  • üìù Code Quality: Professional PR workflow with detailed descriptions
  • ‚ö° Efficiency: Feature branches + PRs enable safe, reviewable changes

üîß Technical Implementation

  • tools/mcp_server.py (lines 768-1165): Detection and integration
    • _detect_existing_codebase() - 15+ project types
    • _detect_task_intent() - keyword-based mode detection
    • autonomous_build_and_deploy() - integrated detection and warnings
  • tools/orchestrator_prompt.txt: Enhancement-aware phases
    • Mode detection section (lines 49-70)
    • Phase 0: Codebase Analysis (lines 71-175)
    • Scout enhancement guidance (lines 257-295)
    • PHASE 2.5 targeted modifications (lines 481-514)
    • Deploy PR workflow (lines 1237-1407)
  • test_detection.py: Standalone validation script

üìö Use Cases

# Fix a bug in existing codebase

Key Insight: Enhancement mode transforms Context Foundry from a greenfield development tool into a comprehensive code evolution platform. It can now fix bugs, add features, upgrade dependencies, refactor code, and add tests to ANY existing project - whether built by Context Foundry or not.
